## 1. Cieľ a motivácia

Pred Fázou 3 (anotácie) a Fázou 5 (cloud sync) potrebujeme vyriešiť otázku:

> **Ako reprezentujeme bod / rozsah v knihe tak, aby zostal stabilný naprieč
> zariadeniami, zmenami fontu / margins, a prípadnou zmenou EPUB súboru?**

Bez stabilnej, **reflow-nezávislej** kotvy nemá zmysel stavať highlighty ani sync —
po prvej zmene veľkosti písma alebo prenose knihy na druhé zariadenie sa všetky
pozície rozsypú a hodnota produktu sa stratí.

V ADR 0001 (M1) sme zvolili `epub_view` ako renderer kvôli stabilite a kvalite
vykresľovania, ale **nehodnotili sme aký lokátor knižnica vie poskytnúť**. Tento
spec zatvára tú medzeru.

Strategy doc `docs/strategy/2026-06-21-renderer-anchor-sync-analysis.md` (sekcia 2)
identifikoval túto medzeru ako blokujúce riziko pre celý sync. M2.5 je odpoveď.

## 4. Anchor formáty

### 4.1 `ReadingPositionAnchor`

```dart
class ReadingPositionAnchor {
  final String chapterId;     // primárny: manifest href; fallback: spine index ako string
  final double progress;      // 0.0 – 1.0 v rámci kapitoly
}
```

**JSON:**
```json
{"c": "OEBPS/Text/chapter01.xhtml", "p": 0.42}
```

**Resolve:**
1. Nájdi `chapterId` v manifest-e otvoreného EPUB-u.
2. Prepni na kapitolu.
3. Scroll na `progress * chapter_render_height` (vykonáva to renderovacia vrstva,
   nie `AnchorCodec` — anchor je iba dáta).

**Sync merge:** `max(progress)` per (book, chapter) — posledná pozícia vyhráva.
Pri konflikte medzi zariadeniami: zariadenie ktoré prečítalo viac určí pozíciu.

### 4.2 `HighlightAnchor`

```dart
class HighlightAnchor {
  final String chapterId;     // rovnaký formát ako vyššie
  final String quote;          // exact text snippet, normalised, max ~200 znakov
  final String prefix;         // 32 znakov pred quote (context)
  final String suffix;         // 32 znakov za quote (context)
  final int? charOffset;       // hint: očakávaná pozícia v kapitole (po normalizácii)
}
```

**JSON:**
```json
{
  "c": "OEBPS/Text/chapter01.xhtml",
  "q": "Down, down, down. Would the fall never come to an end?",
  "pre": "There was nothing else to do, so Alice soon began ",
  "suf": " 'I wonder how many miles I've fallen by this time?'",
  "o": 1247
}
```

**Limity:**
- `quote` max 200 znakov (väčšie selekcie sú zriedka praktické a zaťažujú sync)
- `prefix` + `suffix` po 32 znakov (dostačujúce na disambiguáciu vo väčšine prípadov)
- `charOffset` je *hint* nie nárok — anchor je validný aj bez neho

### 4.3 Voľba `chapterId`

**Primárne:** `manifest href` (napr. `OEBPS/Text/chapter01.xhtml`)
- Stabilné medzi rôznymi súbormi tej istej knihy (pokiaľ vydavateľ nepremenuje)
- Spine index sa môže meniť keď vydavateľ pridá/odstráni front-matter alebo
  reorganizuje
- Konzistentné s tým ako `epub_view` aj `epubx` knihy navigujú

**Fallback:** `spine index` ako stringifikované číslo (`"3"`)
- Použije sa keď manifest href nie je k dispozícii alebo sa nezhoduje
- Indikované pridaním prefixu: `chapterId: "#3"` (hash + index)

### 4.4 Normalizácia textu (`AnchorCodec.normalize`)

Pre konzistentné porovnávanie `quote`/`prefix`/`suffix` s textom kapitoly:

1. **Unicode NFC** (composed forms — `é` ako jeden codepoint, nie `e` + combining acute)
2. **Whitespace collapse** — všetky sekvencie whitespace (vrátane no-break-space,
   tabulátorov, newlines) → jediná medzera
3. **Trim** — odstránenie leading/trailing whitespace
4. **Strip soft hyphens** (U+00AD) — niektoré EPUB-y ich vkladajú pre hyphenation

Nie:
- Lowercase (case je sémanticky relevantný v citátoch)
- Diakritika strip (slovenské `š/č/ž` musia byť verne zachované)
- Punctuation strip (rozdiel medzi „Hello." a „Hello!")

## 5. `AnchorCodec` API

### Súborová organizácia

```
lib/domain/anchor/
├── reading_position_anchor.dart   (data class + toJson/fromJson)
├── highlight_anchor.dart          (data class + toJson/fromJson)
└── anchor_codec.dart              (createHighlight, findHighlight, normalize)
```

Anchor patrí do `lib/domain/` — čisté Dart triedy bez Flutter / Drift / DB
závislostí. Závisí len od `epubx` (parser knihy) a `dart:core`.

### Verejné API

```dart
class AnchorCodec {
  /// Vytvorí HighlightAnchor zo známej selekcie v kapitole.
  /// `selectionStart` a `selectionEnd` sú character offsets v NORMALIZED texte.
  static HighlightAnchor createHighlight({
    required String chapterId,
    required String chapterText,
    required int selectionStart,
    required int selectionEnd,
  });

  /// Resolve: nájdi anchor v aktuálnom obsahu kapitoly.
  /// Vracia TextRange (start, end) v normalised texte, alebo null ak sa text
  /// už nedá lokalizovať.
  static TextRange? findHighlight({
    required HighlightAnchor anchor,
    required String chapterText,
  });

  /// Public helper — useful pre testy aj UI vrstvu.
  static String normalize(String text);
}
```

`TextRange` je už definovaný v `dart:ui` (used by Flutter text fields) — použijeme
ten.

### Resolve algoritmus (`findHighlight`)

1. **Normalize** `chapterText`.
2. **Exact match** — hľadaj `anchor.quote` v okne ±200 znakov okolo
   `anchor.charOffset` (alebo celý text ak `charOffset` chýba).
3. Ak **jeden výskyt** → vráť ho.
4. Ak **viac výskytov** alebo žiadny v hint okne:
   - Skús `anchor.prefix + anchor.quote + anchor.suffix` (presný kontext)
   - Ak match → vráť pozíciu `quote` vo vnútri match-u
5. Ak **stále žiadny exact match**:
   - **Fuzzy match** cez Levenshtein ratio. Implementácia: SequenceMatcher-like
     algoritmus, threshold `ratio ≥ 0.92`
   - Okno: ±500 znakov okolo `charOffset` (širšie ako exact)
   - Ak match → vráť pozíciu
6. Ak ani fuzzy nepomôže → vráť `null`.

Volajúca UI vrstva pri `null` zobrazí napr. „Táto kotva sa stratila po zmene
obsahu knihy" a používateľ môže anotáciu znova vytvoriť alebo zmazať.

### `createHighlight` implementácia

```dart
static HighlightAnchor createHighlight({...}) {
  final normalized = normalize(chapterText);
  final quote = normalized.substring(selectionStart, selectionEnd);
  final prefixStart = math.max(0, selectionStart - 32);
  final suffixEnd = math.min(normalized.length, selectionEnd + 32);
  return HighlightAnchor(
    chapterId: chapterId,
    quote: quote,
    prefix: normalized.substring(prefixStart, selectionStart),
    suffix: normalized.substring(selectionEnd, suffixEnd),
    charOffset: selectionStart,
  );
}
```

## 6. Testovacia stratégia

**Unit testy nad reálnymi EPUB fixtures** v `test/fixtures/`:

1. **Round-trip:** vytvor anchor zo selekcie z `alice.epub` kapitoly → encode JSON
   → decode → `findHighlight` v tej istej kapitole → vráti pôvodný `TextRange`.

2. **Tolerance malých zmien:** vezmi text kapitoly, vlož 5 znakov pred quote,
   `findHighlight` cez fuzzy stále vie nájsť (s posunutou pozíciou).

3. **Disambiguácia:** quote ktorý sa v kapitole vyskytuje 2× (umelo:
   napr. spoločné slovo „the king said"), prefix + suffix správne určia
   ktorý výskyt.

4. **Strata kotvy:** quote vymazaný z textu → `findHighlight` vráti `null`, žiadny
   crash.

5. **Diakritika v `divina.epub`:** vytvor anchor nad reťazcom obsahujúcim
   talianske akcenty (`è`, `à`, `ò`), overí že NFC normalizácia funguje na encode
   aj na resolve.

6. **Normalizácia whitespace:** anchor vytvorený nad textom s mixed whitespace
   (newlines + spaces) sa resolve-ne aj keď chapter text má whitespace inak.

Cieľová coverage: ≥ 90 % v `lib/domain/anchor/`.

## 7. ADR 0002

Spec sa zafixuje v `docs/adr/0002-anchor-format.md`. ADR obsahuje:

- Kontext (rovnaký ako sekcia 1 tu, skrátený)
- Tabuľku z investigácie (sekcia 3.3)

- Dôvod (prečo dva osobitné, prečo href primárne, prečo fuzzy threshold 0.92)
- Dôsledky:
  - Fáza 3 (anotácie) ukladá `HighlightAnchor` ako JSON v `annotations` tabuľke
  - Fáza 5 (sync) prenáša anchor v event log entries
  - Reading position v M3 použije `ReadingPositionAnchor` namiesto `(chapterIndex, progressInChapter)` zo súčasnej schémy
  - Po M3 môže byť potrebná **DB migrácia** `reading_positions` — chápeme to ako akceptovateľný cost
- Alternatívy zamietnuté (CFI via WebView — odmietnuté pre crash na Windows;
  paragraph index — odmietnuté pre nestabilitu pri reflow)
