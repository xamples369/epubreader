## 1. Cieľ

Definovať deterministický, content-based identifikátor knihy ktorý:
- je **stabilný cez re-zip** toho istého obsahu (rôzne ZIP timestamps / compression),
- je **rôzny pre rôzne vydania** (iný preklad, doplnený predhovor, iné kapitoly),
- vie sa spočítať **offline z lokálneho EPUB súboru** bez internetu,
- vystupuje ako **predpoklad pre cross-device sync** (Fáza 5) — anchor sa
  resolve-uje len v rámci knihy s rovnakým identifikátorom.

Strategy doc §7.4: „Definuj book-identity (hash obsahu / normalizovaný id) —
ešte pred Fázou 4/5."

ADR 0002 §4.3 (chapterId scope) stanovuje že anchor sa resolve-uje len v EPUB
súbore s rovnakým content hash-om. M2.6 tento hash dodáva ako konkrétny algoritmus.

## 3. Hash schéma (REVISED — content-only, spine iteration, frozen normalize)

**Algoritmus:** SHA-256 hex-encoded (64 znakov, [0-9a-f]+).

**Scheme version:** `BookIdentity.schemeVersion = 1` (verejná konštanta).
Anchory (ADR 0002) sa scopujú na `(bookHash, schemeVersion)` — bumpnutie tejto
verzie je explicitná migrácia, nie tichá zmena.

**Vstup pre SHA-256** (UTF-8):

```
SCHEME:1
ITEM:<href1>:<frozen normalized text>
ITEM:<href2>:<frozen normalized text>
...
```

**Detaily:**
- Polia oddelené `\n`.
- Žiadne TITLE/AUTHORS/LANG metadata (P3 fix — pridávali krehkosť za nulový
  rozlišovací zisk; iné vydania sa líšia už v obsahu kapitol).
- `<href>` = manifest href zo spine itemref-u. Žiadny `#$i` fallback —
  neresolvnuteľný idref sa skipuje gracefully.
- Iterácia: **`book.Schema.Package.Spine.Items` (vrátane `linear="no"`)**
  (P2 fix — pôvodné `book.Chapters` vynechávalo podkapitoly).
- Href → raw HTML: `book.Content?.Html?[href]` s fallback variantmi pre
  OPF-relative vs root-relative cesty.
- `<frozen normalized text>` = výstup `_identityNormalize` (xhtml parse →
  textContent → whitespace collapse → trim). **NIE** `CanonicalChapterText`
  z M2.5 (P1 fix — anchor extractor je živý kód, identita musí byť zamrznutá).

## 4. Stabilita a unikátnosť (REVISED)

### Stabilný (rovnaký hash)
- Re-zip toho istého obsahu (iné timestamps, iný compression): ✅ (hash čerpá z manifest + content map, nie ZIP metadata)
- Rovnaká kniha, iný file path: ✅
- Rovnaká kniha, iný OS / endianness: ✅ (SHA-256 + UTF-8 sú platform-agnostic)
- **Opravený typo v názve / metadata pri rovnakom obsahu: ✅** (metadata nie sú v hash-i — P3 fix)
- Zmenená obálka pri rovnakom texte: ✅ (cover binary nie je v hash-i)

### Rôzny (rôzny hash)
- Iné vydanie tej istej knihy (iný preklad, iné kapitoly): ✅
- Doplnená kapitola alebo predhovor: ✅
- Drobné editácie textu (oprava preklepov, iné zalomenie): ✅ — chápeme to ako iné vydanie
- Soft hyphen / Unicode-forma sa líši: ✅ (frozen `_identityNormalize` ich neabsorbuje — viac stability, akceptované)

### Akceptované limitácie
- **Image-only / fixed-layout knihy** (komiks, obrázková detská kniha) majú prázdny extracted text → všetky takéto knihy by mali rovnaký hash a kolidovali by. Pre textovú čítačku zameranú na klasickú literatúru irelevantné. Ak by sa to v budúcnosti stalo problémom, schemeVersion bumpneme a do hash-u zahrnieme aj manifest image binary digests.
- **Cross-edícia nie je cieľ** — ten istý titul v dvoch rôznych vydaniach má rôzne hashe (zhodne s ADR 0002 §4.3).

## 5. Edge cases (REVISED)

### Knihy bez akéhokoľvek obsahu
`book.Schema?.Package?.Spine` alebo `Manifest` chýba → `_spineHrefs` vráti
prázdny zoznam → canonical input je len `SCHEME:1`. Hash sa stále spočíta
deterministicky. Invariant „compute vždy vráti 64-char hex" platí.

### Spine itemref bez manifest matchu (malformed EPUB)
`hrefById[idRef]` vráti null → item sa **gracefully skipuje** (žiadny throw,
nie je v hash input-e). Akceptované — malformed EPUB-y sú edge case mimo
cieľa, ale nepadáme.

### Href ktorý sa neresolvuje v `book.Content.Html`
`_resolveHrefToHtml` vráti null pri:
- žiadna zhoda v map-e
- ambiguous suffix match (viac kandidátov) → radšej null než guess

Item sa zarátame ako `ITEM:<href>:` (prázdny text). To znamená že hash je
*aspoň* závislý od počtu a poradia spine items aj keď ich content nevieme
získať. Test `no spine item suffers lookup-failed` toto zachytí pre naše
4 fixtures; ak by sa v budúcnosti vyskytlo v reálnej knihe používateľa,
vidíme to v diagnostike (`debugResolveStatus`).

### Empty content wrappers (cover stránky, oddeľovače)
Niektoré spine itemy (typicky cover wrapper okolo obrázku) majú validný HTML
content ale po extrakcii textu sú prázdne. Toto je legitímne — wrapper
prispieva `ITEM:<href>:` (s prázdnym textom). Test `majority of items resolve
to ok` zabezpečuje že podiel prázdnych nie je dramatický.

### Korupčný EPUB
Ak `EpubReader.readBook` zlyhal pri prvotnom parsovaní, k `BookIdentity.compute`
sa ani nedostaneme. `compute` samotné nikdy nehádže — všetky polia sú
nullable-safe.

### Veľmi veľké knihy
SHA-256 streaming nie je problém — `crypto` zvládne. Canonical input pre
ľudovo dlhé knihy je v rádu stoviek KB až jednotiek MB (P&P: 733KB, Divina:
547KB). Akceptujeme.

## 6. API

```dart
// lib/domain/book_identity/book_identity.dart
class BookIdentity {
  BookIdentity._();

  /// Verzia hash schémy. Nikdy nemeň bez koordinovanej migrácie.
  static const int schemeVersion = 1;

  /// Vráti 64-char hex SHA-256 hash. Nikdy nevyhadzuje výnimku.
  static String compute(EpubBook book);

  /// Diagnostický helper — vráti raw canonical input pred hash-ovaním.
  /// Užitočné na overenie pokrytia (length sanity check pre divina test).
  static String debugCanonicalInput(EpubBook book);
}
```

Žiadny `BookId` value-class wrapper — string je dosť. Wrapper môžeme pridať
neskôr ak by sme chceli typovú bezpečnosť pri prechode cez vrstvy.

## 7. Testovacia stratégia (REVISED — 12 testov)

```
test/domain/book_identity/book_identity_test.dart
```

Testy nad 4 EPUB fixtures (alice, pride, frankenstein, divina):

1. **Determinizmus:** `compute(book)` dvakrát = rovnaký string.
2. **Formát:** výstup je 64 znakov, len `[0-9a-f]`.
3. **`schemeVersion` konštanta existuje a je 1.**
4. **Canonical input začína `SCHEME:1\n`.**
5. **Unikátnosť:** 4 fixtures = 4 rôzne hash-y.
6. **Stabilita cez re-load:** otvor súbor → compute → znovu otvor → rovnaký hash.
7. **Edge: empty EpubBook** — nepadá.
8. **P2 regression guard — divina canonical input > 200K chars** (zachytí
   regresiu späť na `book.Chapters` iteration ktorá by stratila ~95% obsahu).
9. **Alice canonical input > 50K chars** (sanity).
10. **Content-only — vstup neobsahuje `TITLE:`/`AUTHORS:`/`LANG:` markery** (P3).
11. **No lookup-failed na žiadnom fixture** — strict resolve cascade test
    (zachytí ambiguity v `_resolveHrefToHtml`).
12. **Majority of items resolve to ok** — vylúči agresívne HTML stripping.
