## 1. Cieľ

Definovať deterministický, content-based identifikátor knihy ktorý:
- je **stabilný cez re-zip** toho istého obsahu (rôzne ZIP timestamps / compression),
- je **rôzny pre rôzne vydania** (iný preklad, doplnený predhovor, iné kapitoly),
- vie sa spočítať **offline z lokálneho EPUB súboru** bez internetu,
- vystupuje ako **predpoklad pre cross-device sync** (Fáza 5) — anchor sa
  resolve-uje len v rámci knihy s rovnakým identifikátorom.

Strategy doc §7.4: „Definuj book-identity (hash obsahu / normalizovaný id) —
ešte pred Fázou 4/5."

ADR 0002 §4.3 (chapterId scope) stanovuje že anchor sa resolve-uje len v EPUB
súbore s rovnakým content hash-om. M2.6 tento hash dodáva ako konkrétny algoritmus.

## 3. Hash schéma (REVISED — content-only, spine iteration, frozen normalize)

**Algoritmus:** SHA-256 hex-encoded (64 znakov, [0-9a-f]+).

**Scheme version:** `BookIdentity.schemeVersion = 1` (verejná konštanta).
Anchory (ADR 0002) sa scopujú na `(bookHash, schemeVersion)` — bumpnutie tejto
verzie je explicitná migrácia, nie tichá zmena.

**Vstup pre SHA-256** (UTF-8):

```
SCHEME:1
ITEM:<href1>:<frozen normalized text>
ITEM:<href2>:<frozen normalized text>
...
```

**Detaily:**
- Polia oddelené `\n`.
- Žiadne TITLE/AUTHORS/LANG metadata (P3 fix — pridávali krehkosť za nulový
  rozlišovací zisk; iné vydania sa líšia už v obsahu kapitol).
- `<href>` = manifest href zo spine itemref-u. Žiadny `#$i` fallback —
  neresolvnuteľný idref sa skipuje gracefully.
- Iterácia: **`book.Schema.Package.Spine.Items` (vrátane `linear="no"`)**
  (P2 fix — pôvodné `book.Chapters` vynechávalo podkapitoly).
- Href → raw HTML: `book.Content?.Html?[href]` s fallback variantmi pre
  OPF-relative vs root-relative cesty.
- `<frozen normalized text>` = výstup `_identityNormalize` (xhtml parse →
  textContent → whitespace collapse → trim). **NIE** `CanonicalChapterText`
  z M2.5 (P1 fix — anchor extractor je živý kód, identita musí byť zamrznutá).

## 4. Stabilita a unikátnosť

### Stabilný (rovnaký hash)
- Re-zip toho istého obsahu (iné timestamps, iný compression level): ✅ (hash ignoruje ZIP metadata)
- Rovnaká kniha, iný file path: ✅ (hash z obsahu, nie path)
- Rovnaká kniha, iný OS / endianness: ✅ (SHA-256 + UTF-8 sú platform-agnostic)

### Rôzny (rôzny hash)
- Iné vydanie tej istej knihy (iný preklad, iné kapitoly): ✅ (rôzny obsah)
- Doplnená kapitola alebo predhovor: ✅
- Drobné editácie textu (oprava preklepov, iné zalomenie): ✅ — chápeme to ako iné vydanie
- Zmenená obálka (rovnaký text): ⚠️ rovnaký hash! Cover nie je v hash-i. Akceptovaná limitácia — cover nie je súčasť čítacieho obsahu.
- Zmenené metadata (názov, autor): rôzny hash (obe sú v hash-i)

## 5. Edge cases a fallback

### Knihy bez metadát
Ak chýba title, author aj language → polia ostanú prázdne (`TITLE:`, `AUTHORS:`,
`LANG:`). Hash sa stále spočíta nad obsahom kapitol. Bez kapitol → hash by bol
spočítaný nad len `TITLE:\nAUTHORS:\nLANG:\n` — extrémne nepravdepodobné, ale
deterministické.

### Žiadne kapitoly
Ak `book.Chapters` je null alebo prázdny → hash sa spočíta len z metadát.
Vrátime hash (nie throw) — invariant „compute vždy vráti string" je užitočný
pre volajúcich.

### Korupčný EPUB
Ak `EpubReader.readBook` zlyhal pri prvotnom parsovaní, k `BookIdentity.compute`
sa ani nedostaneme. Compute samotné by nemalo throw-nuť — všetky polia sú
nullable-safe.

### Veľmi veľké knihy
SHA-256 streaming nie je problém — `crypto` package zvládne aj GB súbory. Vstup
je v podstate konkatenovaný text, môže byť ~5-50 MB pre dlhú knihu. Akceptujeme.

## 6. API

```dart
// lib/domain/book_identity/book_identity.dart
class BookIdentity {
  BookIdentity._();

  /// Verzia hash schémy. Nikdy nemeň bez koordinovanej migrácie.
  static const int schemeVersion = 1;

  /// Vráti 64-char hex SHA-256 hash. Nikdy nevyhadzuje výnimku.
  static String compute(EpubBook book);

  /// Diagnostický helper — vráti raw canonical input pred hash-ovaním.
  /// Užitočné na overenie pokrytia (length sanity check pre divina test).
  static String debugCanonicalInput(EpubBook book);
}
```

Žiadny `BookId` value-class wrapper — string je dosť. Wrapper môžeme pridať
neskôr ak by sme chceli typovú bezpečnosť pri prechode cez vrstvy.

## 7. Testovacia stratégia

```
test/domain/book_identity/book_identity_test.dart
```

Testy nad 4 EPUB fixtures (alice, pride, frankenstein, divina):

1. **Determinizmus:** `compute(book)` dvakrát = rovnaký string.
2. **Unikátnosť:** 4 fixtures = 4 rôzne hash-y (žiadny náhodný collision).
3. **Formát:** výstup je 64 znakov, len `[0-9a-f]`.
4. **Stabilita cez „re-load":** otvor súbor → compute → znovu otvor → compute → rovnaký hash.
5. **Edge: kniha bez chapters** (synthetic empty EpubBook): vráti hash, nepadá.
6. **Edge: kniha bez title/author** (synthetic): vráti hash, nepadá.

## 8. ADR 0003 outline

`docs/adr/0003-book-identity.md`:

- Kontext (rovnaký ako §1)
- Voľba algoritmu (SHA-256) a vstupu (normalised content per §3)
- Alternatívy zamietnuté:
  - **SHA-256 celého ZIP súboru** — citlivé na re-zip metadata
  - **Hash len kanonického textu** — stratíme schopnosť rozlíšiť „rovnaký text, iná kniha" (nová obálka, nový predhovor s rovnakým hlavným textom)
  - **OCN / ISBN extrakcia z metadát** — nestabilné (chýba v Gutenberg knihách), nedeterministické
  - **xxHash / Blake2** — rýchlejšie ale netreba pre náš use case; SHA-256 je všade dostupný a štandard
- Dôsledky pre M3 (DB column kedykoľvek) a Fázu 5 (anchor scope = same book hash)
